Delivery

Emergency reference · current as of 2026-04-22

What it is

A web app used in the warehouse to check in the daily produce delivery. The delivery worksheet (a PDF from IT) is uploaded each morning; warehouse staff walk the floor marking items as received/short/over, flagging pulls destined for the retail floor, and reviewing a 3-day high-count forecast. Exceptions are captured for the buyer at end of day.

If it's broken — start here

  1. Site is down → GCP console → Cloud Run → delivery-app → Revisions tab. Roll back to the last known-good revision if the latest is red.
  2. Today's worksheet won't load → Firebase console → Storage bucket delivery-worksheet-app.firebasestorage.app, path delivery-files/incoming/. Confirm today's file arrived.
  3. Data missing / parse looks wrong → Firestore → deliveries collection. Compare the Firestore doc to the source PDF; re-upload the PDF to trigger re-parse if needed.
  4. Deploy failed → GCP console → Cloud Build → History. Typical fix: re-run ./deploy.sh from a clean checkout.
