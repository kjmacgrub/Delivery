What it does

Brief: Daily produce check-in web app for the warehouse team.

General: An admin uploads the daily delivery worksheet (PDF from IT) to Firebase Storage. A Python backend parses it with pdfplumber, extracts supplier blocks and line items (including floor-pull quantities flagged with an asterisk), and writes a structured record to Firestore. Warehouse staff open the app on phones/iPads, walk the supplier sections, and mark each item as received/short/over/return, optionally confirming pull quantities. A separate daily CSV carries basement/floor location metadata, and a second PDF carries a 3-day high-count forecast — both are parsed and joined to the worksheet. At completion, an exception report listing discrepancies + pulls is snapshotted for the buyer. (An IT-driven consolidation to a single combined CSV is in planning — see NEW_CSV_FORMAT.md.)

Develop & deploy

Setup (once)python3 -m venv .venv && .venv/bin/pip install -e .
Run locally.venv/bin/python -m uvicorn delivery.api.app:create_app --factory --host 0.0.0.0 --port 8001 --reload
Local URLhttp://localhost:8001 (8000 is reserved for a different app)
Tests.venv/bin/pytest — tests in tests/ (optional dev dep)
Deploy./deploy.sh — builds Docker image via Cloud Build, deploys new Cloud Run revision, routes 100% traffic.
Version displayUI shows v1.XX · <commit-hash> from VERSION_HISTORY in app.js; cache-busters auto-injected.
